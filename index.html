CSS - BASICS
  1. Css stands for "Cascading" "Style" "Sheet"
    • "Style" - A single component of design like width, height, color etc. called "property"
    • "Sheet" - A group of styles
    • "Cascading" - Properties cascade down from multiple selectors from internal & external css and inline-styles, and finally applied based on the order of evaluation.
  2. Order of Evaluation - When css encounters confliting property from mutiple sources, it applies the order of evaluation to choose one of the options.
    1. Although the evaluation logic has multiple rules, the simplest way to understand is the property with most-specific reference to the particular element would be applied.
    2. For ex. All students of the college 'X' must wear white-shirt. Students from IT branch of 'X' must wear blue shirt. Sagar from IT banch college 'X' would wear black shirt. Then Sagar would wear black shirt while other IT student would wear blue and the rest of the college would wear white.
    3. Add the !important tag to the property (like width: 100px !important;), if you want to override this behaviour of evaluation and force that selected property to apply in case of conflicts. If muptiple properties with important tag conflicts, the order of evaluation is applied between the important-properties only.
  3. Basic structure of Properties - PropertyName: Value Unit ; For ex. background-color:#cccccc; width:100px;
  4. Selectors
    • Simple - Universal *{}, Element div{}, Id #id{}, Class .heading{}, Attribute section[id="profile"]{}
    • Combinator - Descendant section div{}, Child section>div{}
    • UserAction - for ex. selector:hover{} add properties upon user-actions like hover, focus etc. We used it on the button hover to change its background-color.
    • Pseudo Element - Adds content to the page via css. You can easily spot them with '::' as in 'body::after{}'. Remember the .clearfix class that was added to clear the effect of float. It used the ::after pseudo-element selector.
  5. Css-Box-Model
    • Every element is a rectangle with 4 primary components - dimension(width/height), padding(inside-the-box), border(outside-the-box), margin(outside-the-box)
    • final-dimension = given-dimension + padding + border
    • box-sizing:border-box includes padding and border within the given-dimensions and helps us avoid calculations.
    • Use the chrome inspector to visualize the box-model for any element. It helps you indentify the source of problem, if the dimesions are not renedering properly while Htmlizing the designs.

FLOAT & CLEAR -
Summary -
  1. Float property is to wrap text around the images just like print-media (newspaper/magzines)
  2. Since float can align elements next to each other, it started to be used as a hacky solution for developing skeletons.
  3. Order of elements in which they are floated matters. If you change it the design wold be impacted.
  4. The effect of float trickles to the subsequent elements as well. 'Clear' property is used get rid of this trickling effect from the previous elements.
  5. Float may impact the height of the container. If all elements inside a container are floated, the container finds no element inside it and loses its height. In order to retain the height of the contain, atleast one element has to be placed(sacrificed) with 'clear' property having no-floating effect. The .clearfix class with the pseudo-element::after selector is a much elegant solution to handle this.
  6. DO NOT USE FLOAT for making column-skeletons now. USE FLEX INSTEAD.

FLEXBOX -
Summary -
  1. Flex are native css solution to build column layout
  2. Flex can also be used for achieving responsiveness
  3. Flex has two basic rules -
    1. Parent-Child Relationship - The parent often referred to as 'container' sets the rules for all it's children often called flex-items.
    2. Unidrectional - All flex-items can only be aligned in one-direction only i.e row or columns.
  4. The flex-direction determines the main-axis and cross-axis is properteis on which properties like align-content, justify-content work.
  5. In flex properties,
    1. 'content' refers to the extra space left in the container around the items
    2. 'justify' refers to applying the property on the main-axis.
    3. 'align' refers to applying the property on the cross-axis.
  6. If not using any css-frameworks like bootstrap or tailwind, USE FLEX for making column-skeletons now.
  7. Refer for flexbox properties - https://css-tricks.com/snippets/css/a-guide-to-flexbox/

CSS MEDIA QUERIES -
Summary -
  1. Media queries are used to add classes specific to a partifuclar range of screen resoultion.
  2. Media queries are used to enable responsiveness.
  3. Responsive design is a design that can adjust to all screens i.e. mobiles, tablets and desktops.
  4. Responsive Designing = Manage Resizing + Manage Repositioning
  5. For ex, the following would be applied to body when width is less than 640px.
    @media only screen and (max-width: 640px) { body{ padding: 0px 30px; } }
